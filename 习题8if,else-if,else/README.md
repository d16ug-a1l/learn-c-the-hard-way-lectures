# 习题8if,else-if,else  

语法格式如下：
```c
/*
 * 需要注意的是程序控制流是从上到下的，先从最上面的条件进行判断
 * 条件成立之后执行完成区域内的代码，后续代码就不执行了
 */

if (判断条件) {
    条件为真时执行的代码;
} else if (判断条件) {
    条件为真时执行代码;
} else {
    以上条件都不满足时执行的代码;
}

```  

这段反汇编结果的核心变化是**引入了局部变量，并增加了三个带有 `if` 条件判断的控制流**。它的 C 语言源码功能是：**定义了三个局部变量（分别赋值为 0、2、-1），并用 `if` 分别判断它们是否为 0。如果非 0 则调用 `puts` 打印，如果为 0 则跳过。**

以下是关键变化的深度拆解，帮你彻底看懂局部变量在栈上的布局以及 `if` 语句在汇编层面的实现：

---

### 1. 栈空间变化（开辟了更多空间）

```assembly
0x00001151 <+8>:     sub    $0x20,%rsp

```

* **变化：** 这次栈指针 `rsp` 减去了 **`0x20`（十进制 32 字节）**，而上一次只有 `0x10`。这是因为代码里多了 3 个整型（`int`）局部变量，每个占 4 字节，需要更大的栈空间。

---

### 2. 参数与局部变量的栈布局

因为是未优化的代码（`-O0`），我们可以通过相对于 `rbp` 的偏移量，清晰地还原出栈上的内存布局：

```assembly
0x00001155 <+12>:    mov    %edi,-0x14(%rbp)  ; argc 存入 rbp-20 的位置
0x00001158 <+15>:    mov    %rsi,-0x20(%rbp)  ; argv 存入 rbp-32 的位置

```

接下来是关键，**三个局部变量被初始化并放入栈中**：

| 汇编指令 | 对应栈地址 | 写入的数值 | 对应的 C 变量 |
| --- | --- | --- | --- |
| `movl $0x0,-0xc(%rbp)` | `rbp - 12` | `0` | `int a = 0;` |
| `movl $0x2,-0x8(%rbp)` | `rbp - 8` | `2` | `int b = 2;` |
| `movl $0xffffffff,-0x4(%rbp)` | `rbp - 4` | `-1` (`0xffffffff` 是 -1 的补码) | `int c = -1;` |

---

### 3. 解密三个 `if` 条件跳转控制流

汇编通过 **`cmpl` (比较)** + **`je` (若相等则跳转)** 的组合来实现 `if (变量 != 0)` 的逻辑：

#### 🔴 第一个判断：`if (a != 0)` —— 结果为假，跳过打印

```assembly
0x00001163 <+26>:    cmpl   $0x0,-0xc(%rbp)     ; 将变量 a (栈 rbp-12) 与 0 比较
0x00001167 <+30>:    je     0x1178 <main+47>    ; 【条件触发】如果 a == 0，直接跳转到 0x1178

```

* **分析：** 因为 `a` 的值确实是 `0`，满足 `je`（Jump if Equal）的条件，程序**直接跳过**了中间的 `puts` 调用，飞跃到了 `0x1178`。所以第一个字符串**不会**被打印。

#### 🟢 第二个判断：`if (b != 0)` —— 结果为真，执行打印

```assembly
0x00001178 <+47>:    movl   $0x2,-0x8(%rbp)     ; 初始化变量 b = 2
0x0000117f <+54>:    cmpl   $0x0,-0x8(%rbp)     ; 将变量 b 与 0 比较
0x00001183 <+58>:    je     0x1194 <main+75>    ; 【条件不触发】如果 b == 0 则跳转

```

* **分析：** 此时 `b` 是 `2`，不等于 `0`。因此 `je` 条件不成立，程序不会跳转，而是**顺序向下执行**下面的打印指令：

```assembly
0x00001185 <+60>:    lea    0xe9b(%rip),%rax    ; 加载字符串地址 (0x2027)
0x0000118c <+67>:    mov    %rax,%rdi
0x0000118f <+70>:    call   0x1050 <puts@plt>   ; 成功调用 puts 打印！

```

#### 🟢 第三个判断：`if (c != 0)` —— 结果为真，执行打印

```assembly
0x00001194 <+75>:    movl   $0xffffffff,-0x4(%rbp) ; 初始化变量 c = -1
0x0000119b <+82>:    cmpl   $0x0,-0x4(%rbp)        ; 将变量 c 与 0 比较
0x0000119f <+86>:    je     0x11b0 <main+103>      ; 【条件不触发】如果 c == 0 则跳转

```

* **分析：** `c` 是 `-1`，不等于 `0`，`je` 再次失效。程序继续顺序向下执行，调用 `puts` 打印第三个字符串（地址 `0x2048`）。

```assembly
0x000011a1 <+88>:    lea    0xea0(%rip),%rax
0x000011a8 <+95>:    mov    %rax,%rdi
0x000011ab <+98>:    call   0x1050 <puts@plt>   ; 成功调用 puts 打印！

```

---

### 总结：它对应的 C 源代码

这段反汇编是由下面这段 C 代码编译出来的：

```c
#include <stdio.h>

int main(int argc, char *argv[]) {
    int a = 0;
    if (a != 0) {       // 汇编中表现为：如果 a == 0 则跳过
        puts("字符串1"); // 对应地址 0x2008 (被跳过了)
    }

    int b = 2;
    if (b != 0) {       // 汇编中表现为：如果 b == 0 则跳过
        puts("字符串2"); // 对应地址 0x2027 (成功打印)
    }

    int c = -1;
    if (c != 0) {       // 汇编中表现为：如果 c == 0 则跳过
        puts("字符串3"); // 对应地址 0x2048 (成功打印)
    }

    return 0;
}

```
