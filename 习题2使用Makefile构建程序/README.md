# 习题2使用Makefile构建程序
---
0. 我们在这一节中使用了一个环境变量设置CFLAGS="-Wall",这个-Wall参数的作用就是启用所有的警告信息,如何获得这个解释呢,现在你可以使用gpt/Gemini等工具来提问,解答你的问题,当然按照教程的要求你需要在man gcc中进行查找-Wall这个参数的作用.
---
1. 我们这一节中编写了一个小的Makefile文件,如何你的项目目录中没有Makefile文件,也可以执行make命令,make命令比如命令`make ex1`,这个命令会自动执行`cc ex1.c -o ex1 `,你可以把make理解成一个经验丰富的开发人员,你只要告诉他生成最终产物的名称,他会在你当前目录中自动去寻找ex1名称的文件,如果存在那么就自动去编译.
---
2. 我们前面使用到了环境变量`CFLAGS="-Wall"`,那么我们如何使用makefile文件去构建呢,下面是我们写的makefile文件:
```makefile
CFLAGS=-Wall -g
all: clean ex1
clean:
    rm -f ex1
```
这是我们写的makefile文件,第一行是我们之前需要添加的环境变量信息,第二行是我们需要使用的构建命令,首先执行clean,那么clean对应的内容就在下面,执行的是`rm -f ex1`,先删除之前的构建产物,然后再去生成最终产物ex1.
